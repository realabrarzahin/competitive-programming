Game (Easy)

This is the easy version of the problem. Both versions are worth the usual 100100100 points. In this version, the major difference is N≤2000N \le 2000N≤2000.

You are playing a game on yet another array AAA of size NNN.

You have KKK coins at the start, and will do the following action:

  • You start with sum=0,x=0sum = 0, x = 0sum=0,x=0.
  • For each i=1,2,…,Ni = 1, 2, \ldots, Ni=1,2,…,N (in this order), choose to do exactly one of the following 333 actions:
    • do nothing, pay no cost.
    • update sum←sum+Aisum \leftarrow sum + A_isum←sum+Ai​, and then update x←x+1x \leftarrow x + 1x←x+1, pay 111 coin.
    • update sum←sum+Ai+xsum \leftarrow sum + A_i + xsum←sum+Ai​+x, and then update x←x+1x \leftarrow x + 1x←x+1, pay 222 coins.

Your goal is to maximize the final value of sumsumsum.

Unfortunately, the starting amount of coins is not fixed. Find the answer for each K=1,2,…,2⋅NK = 1, 2, \ldots, 2 \cdot NK=1,2,…,2⋅N.

Input Format

  • The first line of input will contain a single integer TTT, denoting the number of test cases.
  • Each test case consists of multiple lines of input.
    • The first line of each test case contains NNN - the size of the array.
    • The second line contains NNN integers - A1,A2,…,ANA_1, A_2, \ldots, A_NA1​,A2​,…,AN​.

Output Format

For each test case, output on a new line the maximum values of sumsumsum for each K=1,2,…,2⋅NK = 1, 2, \ldots, 2 \cdot NK=1,2,…,2⋅N

Constraints

  • 1≤T≤5001 \le T \le 5001≤T≤500
  • 2≤N≤20002 \le N \le 20002≤N≤2000
  • 1≤Ai≤N1 \le A_i \le N1≤Ai​≤N
  • The sum of NNN over all test cases does not exceed 200020002000.
